IDEAÇÃO PARA BUILD 1

Estamos explorando tecnologias atuais para a detecção de soft skills durante entrevistas de emprego, ainda assim, esse objetivo não é alcançado apenas por explorar seu foco, e que portanto há benefícios em passar por conteúdos que são essenciais para a construção de uma ferramenta.

Sendo coerente com o aprendizado anterior ( instruction 2 ), implementamos a primeira versão de uma interface não funcional e não acessível no WordPress, a qual consiste nos elementos já organizados dentro do CMS, mas sem conexão com o banco de dados, mas já prontos para serem trabalhados: embora o WordPress seja “clica e arrasta”, ao definir elementos de páginas na plataforma, o código PHP já é gerado, com o uso de plugins adequados, o código CSS também.

IMPLEMENTAÇÃO PARA BUILD 2

Nosso trabalho resultou em:

  • Utilizamos um servidor simulado em máquina local utilizando o XAMPP.
  • Alinhamos a prototipagem digital de acordo com o objetivo, trazendo uma segunda versão, mais robusta.
  • Buscamos plugins gratuitos necessários para a transição da prototipagem digital do Figma para o WordPress em forma de elementos.
  • Remanejamos os elementos dentro do WordPress como a primeira versão da interface não funcional.
  • Definimos as descrições necessárias para cada página do site (já que até então estávamos usando Lorem Ipsum).

Ferramentas utilizadas: Wordpress, Maria DB My SQL, Figma, Xamp

Plugin utilizado: Elementor
